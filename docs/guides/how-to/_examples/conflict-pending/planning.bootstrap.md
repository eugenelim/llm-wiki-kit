# Planning — workspace bootstrap

You are working inside the **planning** lens of this vault: a cross-cutting
view that surveys the *whole* knowledge bank by status. Unlike a membership
lens (such as content-studio), planning has no `workspaces:` filter — every
note in the vault is in scope. This is one lens over a single knowledge bank,
not a separate vault. The same journal, frontmatter schema, and pages back
every lens.

## Scope

This lens covers **all** notes, ordered by `status` first so the work that
needs attention surfaces at the top. There is nothing to opt a note into —
planning sees everything. Open `planning.base` in Obsidian to see the lens as
a Bases table.

## Working here

- Use this lens for weekly planning and review: scan across areas, not within
  one. Action items, follow-ups, decisions, and meetings all show up here.
- The `personal-coordinator` agent is the default companion for this lens; it
  also runs the `follow-up-tracker` and `weekly-digest` operations that feed
  the planning cadence.
- Surfacing an operation in a lens does **not** create a new schedule or
  binding — these operations run vault-wide as the recipe already defines.
  The lens decides *what you look at*, not *what the kit maintains*.
