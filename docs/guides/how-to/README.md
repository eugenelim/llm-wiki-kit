# How-to guides

Task-oriented user docs, per [Diátaxis](https://diataxis.fr/). A how-to
is a recipe for solving one specific problem: the reader already knows
what they want and needs the steps to do it. Not for beginners, not
exhaustive — focused.

## Index

- **[Resolve a conflict](resolve-a-conflict.md)** — the kit wrote a
  `<path>.proposed` sidecar (because it detected your edits) and you
  want to reconcile.

## Other Diátaxis buckets

- **[Tutorials](../tutorials/)** — learning-oriented; the reader is
  new and wants to build a working result.
- **[Reference](../reference/)** — information-oriented; the reader
  is looking something up.
- **[Explanation](../explanation/)** — understanding-oriented; the
  reader wants the *why* behind a design.

The kit's existing user docs still live flat under `docs/guides/*.md`
(`customizing.md`, `sync-options.md`, `web-clipper.md`). Migrating
them into how-to format is a future docs PR, not adoption scope.
