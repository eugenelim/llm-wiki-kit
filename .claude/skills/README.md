# Kit-side skills

Skills agents invoke while developing the kit. Vault-side skills that
ship to users live under `core/files/skills/` and
`templates/*/files/skills/` — those are different and separate (see
[`AGENTS.md` § Two scopes, one repo](../../AGENTS.md#two-scopes-one-repo)).

- [work-loop](work-loop/SKILL.md) — start here for any non-trivial change
- [new-spec](new-spec/SKILL.md) — scaffold a spec + plan in `docs/specs/`
- [new-adr](new-adr/SKILL.md) — record a load-bearing decision in `docs/adr/`
- [new-rfc](new-rfc/SKILL.md) — propose a change in `docs/rfc/`
- [bug-fix](bug-fix/SKILL.md) — fix a defect with root-cause discipline
- [update-conventions](update-conventions/SKILL.md) — open an RFC to amend `CONVENTIONS.md`

The five workflow skills land in PR-2 of RFC-0002. Their links resolve
once that PR merges.
